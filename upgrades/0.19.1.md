## What Changed

### Removed Stale Global Install References

Since v0.17.3, Instar uses shadow installs (`{stateDir}/shadow-install/`) for auto-updates — global installs (`npm install -g instar`) are no longer needed. However, several user-facing locations still recommended global installs:

1. **Installation docs** (`site/src/content/docs/installation.md`): Removed the "Global Install (Optional)" section entirely. `npx instar` is the only recommended installation method.

2. **Setup CLI error messages** (`src/commands/setup.ts`): Two error messages suggested `npm install -g instar` — updated to recommend `npx instar` and reference shadow installs instead.

## What to Tell Your User

No action needed — this is a documentation cleanup. If you previously installed Instar globally, it still works, but auto-updates happen through shadow installs regardless of how you installed.
