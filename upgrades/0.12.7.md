# Instar v0.12.7 Release Notes

## Bug Fixes

### SystemReviewer no longer crashes with TypeError on startup

**What broke:** SystemReviewer was throwing `TypeError: Cannot read properties of undefined (reading 'includes')` at startup in a burst pattern. This happened because the constructor received `disabledProbes: undefined` from the server initialization (when the user hadn't set this config field), and spreading `{ disabledProbes: undefined }` over the defaults overrode the default `[]` with `undefined`.

**What changed:** The constructor now strips `undefined` values from the config object before merging, preserving all defaults for unset fields.

**Action required:** None — this fix applies automatically on upgrade.

### Scheduler now enabled by default on fresh install

**What broke:** `instar init` created a config with `scheduler.enabled: false`, meaning freshly initialized agents had their jobs defined but none running. Git sync, health checks, and other default jobs were silently inactive until someone manually enabled the scheduler.

**What changed:** Fresh `instar init` and the setup wizard now create configs with `scheduler.enabled: true`. Jobs run from the moment the agent starts.

**Action required for existing agents:** If you initialized with v0.12.6 or earlier and want jobs to run, update your `.instar/config.json`:
```json
{
  "scheduler": {
    "enabled": true
  }
}
```
Restart the server after making this change.
