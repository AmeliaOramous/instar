# Upgrade Guide — v0.12.7

<!-- bump: patch -->

## What Changed

Two bug fixes that affect all agents.

**SystemReviewer TypeError fixed.** The SystemReviewer was throwing `TypeError: Cannot read properties of undefined (reading 'includes')` in a burst pattern on startup. Root cause: the constructor spread `{ disabledProbes: undefined }` over the defaults, overriding the default `[]` array with `undefined`. Now the constructor filters out `undefined` values before merging, preserving all defaults.

**Scheduler now enabled by default on fresh install.** `instar init` was creating configs with `scheduler.enabled: false`, meaning freshly initialized agents had jobs defined but none running. Git sync, health checks, and other default jobs were silently inactive until manually enabled. Fresh installs now start with `scheduler.enabled: true`.

## What to Tell Your User

- **SystemReviewer errors are resolved.** "Those TypeError bursts you may have seen from SystemReviewer on startup are fixed."
- **For newly initialized agents**: "Jobs now run from the start. No need to manually enable the scheduler."
- **For existing agents with scheduler disabled**: "If your agent was initialized before this version and you want scheduled jobs to run, set `scheduler.enabled: true` in your `.instar/config.json` and restart the server."

## Summary of New Capabilities

| Fix | What it does |
